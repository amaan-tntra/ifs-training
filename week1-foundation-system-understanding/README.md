# Week 1 – Foundation & System Understanding

This week focused on understanding the basic technical concepts required to work with the system, including IFS architecture, DevOps fundamentals, and SQL practice. The goal was to build a strong foundation before moving to more advanced topics.

---

## Topics Covered

### IFS Technical Architecture
- Overview of the IFS ERP system
- Understanding the multi-layer architecture:
  - Client Layer (User Interface)
  - Application Layer (Business Logic)
  - Database Layer (Oracle Database)
- Understanding how data flows between these layers

### DevOps Fundamentals
- Introduction to DevOps concepts
- Understanding the DevOps lifecycle
- Basic idea of CI/CD (Continuous Integration and Continuous Deployment)
- Importance of automation and collaboration in software delivery

### SQL & PL/SQL Refresher
- Practiced SQL fundamentals
- Writing queries for data retrieval and analysis
- Understanding relationships between tables
- Using joins, filtering, aggregation, and subqueries

## SQL Practice

Two SQL scripts were created:

### 1. schema_setup.sql
Contains database setup scripts including:
- Table creation
- Sample department data
- Sample employee data

Tables created:
- `tntra_departments`
- `tntra_employees`

### 2. practice_queries.sql
Contains various SQL queries to practice and demonstrate SQL concepts such as:

- SELECT queries
- Filtering with `WHERE`
- Sorting with `ORDER BY`
- Aggregation (`COUNT`, `AVG`, `MAX`)
- `GROUP BY` and `HAVING`
- `JOIN` queries
- Subqueries
- Pattern matching with `LIKE`
- Range filtering with `BETWEEN`
- Filtering with `IN`
- Limiting results using `FETCH`
- Analytical queries (salary comparison, department statistics)
