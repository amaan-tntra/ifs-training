# DevOps Notes

## What is DevOps
- DevOps stands for **Development + Operations**.
- It is a set of practices that improves collaboration between development and operations teams.
- The goal is to **deliver software faster, more reliably, and with fewer errors**.
- DevOps emphasizes **automation, continuous integration, and continuous deployment**.

## Why DevOps is Important
- Enables **faster software delivery**
- Improves **team collaboration**
- Reduces **manual deployment work**
- Helps detect and fix **bugs earlier**
- Ensures **stable and reliable applications**

## DevOps Lifecycle

### 1. Development
- Developers write code and implement features.
- Code is stored in **version control systems like Git**.

### 2. Build
- The application is prepared for execution.
- Dependencies are installed and the application is packaged.

### 3. Test
- Automated tests verify that the code works correctly.
- Helps identify bugs before deployment.

### 4. Release
- The tested build is prepared for deployment.
- Versioning and packaging of the application are done.

### 5. Deploy
- The application is deployed to servers or cloud environments.
- Deployment can be automated.

### 6. Monitor
- System performance and errors are monitored after deployment.
- Helps maintain system stability and detect issues quickly.

## CI/CD

### Continuous Integration (CI)
- Developers frequently merge code into a shared repository.
- Each commit triggers automated build and test processes.

### Continuous Deployment / Delivery (CD)
- After successful testing, the application is automatically deployed.
- Enables faster and more consistent software releases.
